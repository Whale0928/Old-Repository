변수 선언 방식

        var  : 변수, 변수명 중복이 가능(중복일때는 먼저 선언된 변수를 덮어쓰기),함수레벨 scope
        let  : 변수, 변수명 중복 X , 블럭{} 레벨 scope
        const: 상수, 변수명 중복 X , 블럭{} 레벨 scope

        1순위 : const
            - js 는 특정 요소를 선택해 사용하는 경우가 많아서 변수에 고정 시켜 둔다.

        2순위 : let

        3순위 : var (ES6 이후부터 사용빈도가 줄어듬.)

JS의 자료형

        자바스크립트는 변수 선언시 별도의 자료형을 지정하지 않는다.
        ->변수에 대입되는 값(리터럴)에 의해서 자료형이 결정됨(자동?)

        -String (문자열)
        -number (숫자) ->정수 / 실수 모두 포함.
        -boolean(t/f)
        -object (객체) -> 배열(Array) 
        -JS객체{K:V , k:v} 
        -function (함수) 
        -undifined(정의되지 않은 변수) -> 변수가 선언만 되고 값이 대입되지 않은 상태.

        (참고 , 타입이라고 하기에는 애매함 값(리터럴)의 개념)
        -null (참조하는게 없다.) -> document.getElementById("오타입력") : 해당요소를 선택 못할 경우
